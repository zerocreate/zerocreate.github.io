1） application.properties，绑定属性值到字段

1.1）基本使用

我们可以把一些属性配置在application.properties文件中，然后在类的字段上通过注解@Value(value=”${config.name}”)，这样就会自动读取该配置文件中的属性值赋值给字段。一般用于配置数据库连接池属性、程序端口、邮箱账号等。

1.3）读取属性值中文乱码

当application.properties文件中配置的属性值包含中文时，在其他地方读取输出结果会是乱码。解决方案分两步：

  • 1 修改文件编码

    依次点击File -> Settings -> Editor -> File Encodings，将Default encoding for properties files设置为UTF-8并勾选Transparent native-to-ascii conversion。

  • 2 重新修改application.properties属性值

    修改完编码后，application.properties中的中文内容会变成乱码，我们改回中文就可以了。

2） application.properties，绑定属性值到类

2.1）基本使用

有时候属性值太多了，一个个绑定到字段上太累。官方提倡绑定一个类，这里我们建一个ConfigBean类，类上面添加注解@ConfigurationProperties(prefix = "com.springboot")。这样application.properties中以com.springboot打头的name属性值和wish属性值会自动绑定到这个类上面。
配置完还需要在SpringBoot入口类加上@EnableConfigurationProperties并指明要加载哪个bean，如果不写ConfigBean.class，在bean类那边添加。

5） 随机值配置

配置文件中${random} 可以用来生成各种不同类型的随机值，从而简化了代码生成的麻烦，例如 生成 int 值、long 值或者 string 字符串。

6） 多环境配置

当应用程序需要部署到不同运行环境时，一些配置细节通常会有所不同。最简单的比如日志，生产日志会将日志级别设置为WARN或更高级别，并将日志写入日志文件；而开发环境下日志级别会设置为DEBUG，日志输出到控制台即可。

如果按照以前的做法，就是每次发布的时候替换掉配置文件，这样太麻烦了。Spring提供多环境配置的支持，我们可以为每个环境编写配置文件，然后在application.properties或命令行中指定要使用哪个环境的配置文件。

在SpringBoot中多环境配置文件名需要满足application-{profile}.properties的格式，其中{profile}对应你的环境标识，比如：

  • application-dev.properties：开发环境
  • application-prod.properties：生产环境

在开发环境下，我们只需要在application.properties中配置spring.profiles.active属性值为dev，程序就会使用对应开发环境。切换到生产环境时，只需要修改application.properties中spring.profiles.active属性值为prod即可。
